Чек-лист ревью по ролям

Этот документ связывает файлы документации с ролями команды, которые должны просмотреть и утвердить их до того, как MVP будет считаться завершённым.

Роли в проекте:

  • Тимлид / DevOps
  • Backend-разработчик
  • Frontend-разработчик
  • DBA (администратор базы данных)
  • Ответственный за документацию

1. Основные продуктовые документы

  • docs/product-overview.md / .ru.md
  • Обязательное ревью:
    • Тимлид / DevOps (общий охват и границы MVP)
    • Backend-разработчик (техническая реализуемость)
    • Frontend-разработчик (предполагаемые аспекты UI / UX)

  • Желательно просмотреть:

    • DBA (влияние на данные и хранение)

  • docs/modules.md

  • Обязательное ревью:
    • Тимлид / DevOps
    • Backend-разработчик

  • Желательно просмотреть:

    • Frontend-разработчик
    • DBA

  • docs/db-schema.md

  • Обязательное ревью:
    • Оба DBA
    • Backend-разработчик
  • Желательно просмотреть:
    • Тимлид / DevOps

2. Руководства

  • docs/user-guide.md / .ru.md
  • Обязательное ревью:
    • Frontend-разработчик (потоки интерфейса должны совпадать с реализацией)
    • Ответственный за документацию

  • Желательно просмотреть:

    • Тимлид / DevOps

  • docs/admin-guide.md / .ru.md

  • Обязательное ревью:
    • Backend-разработчик (ожидания по admin API и модели данных)
    • Тимлид / DevOps

  • Желательно просмотреть:

    • DBA (админские сценарии, влияющие на каталог и справочники)

  • docs/api-overview.md / .ru.md

  • Обязательное ревью:
    • Backend-разработчик
    • Тимлид / DevOps

  • Желательно просмотреть:

    • DBA (паттерны чтения, влияние на запросы к БД)

  • docs/deploy-guide.md / .ru.md

  • Обязательное ревью:
    • Тимлид / DevOps
  • Желательно просмотреть:
    • Backend-разработчик (окружение и настройки)
    • DBA (части, относящиеся к БД)

3. Архитектурные решения (ADR)

  • docs/adr/0000-adr-template.md
  • docs/adr/0001-initial-fastapi-choice.md
  • docs/adr/0002-switch-to-django-drf.md
  • docs/adr/0003-database-schema-and-relations.md
  • docs/adr/0004-search-engine-integration.md

  • docs/adr/0005-single-node-docker-deployment.md

  • Обязательное ревью:

    • Тимлид / DevOps
    • Backend-разработчик
    • DBA (для ADR, связанных с базой данных)
  • Желательно просмотреть:
    • Frontend-разработчик (для решений, влияющих на форму API)

4. Визуальные материалы и ассеты

  • docs/diagrams/ (ER-диаграммы, архитектурные схемы)
  • Обязательное ревью:
    • Тимлид / DevOps
    • DBA (для диаграмм БД)

  • Желательно просмотреть:

    • Backend-разработчик
    • Frontend-разработчик

  • docs/screenshots/

  • Обязательное ревью:
    • Frontend-разработчик
    • Ответственный за документацию

5. Процесс

Перед демо MVP:

  1. Каждая роль проходит по файлам, отмеченным как «Обязательное ревью».
  2. Комментарии оставляются через PR или в виде встроенных заметок.
  3. Ответственный за документацию обновляет как английскую, так и русскую версии, где это применимо.
