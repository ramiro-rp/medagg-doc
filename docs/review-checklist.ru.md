# Чек-лист ревью по ролям

Этот документ связывает файлы документации с **ролями команды**, которые
должны просмотреть и утвердить их до того, как MVP будет считаться завершённым.

Роли в проекте:

- **Тимлид / DevOps**
- **Backend-разработчик**
- **Frontend-разработчик**
- **DBA (администратор базы данных)**
- **Ответственный за документацию**

---

## 1. Основные продуктовые документы

- `docs/product-overview.md` / `.ru.md`
  - **Обязательное ревью:**
    - Тимлид / DevOps (общий охват и границы MVP)
    - Backend-разработчик (техническая реализуемость)
    - Frontend-разработчик (предполагаемые аспекты UI / UX)
  - **Желательно просмотреть:**
    - DBA (влияние на данные и хранение)

- `docs/modules.md`
  - **Обязательное ревью:**
    - Тимлид / DevOps
    - Backend-разработчик
  - **Желательно просмотреть:**
    - Frontend-разработчик
    - DBA

- `docs/db-schema.md`
  - **Обязательное ревью:**
    - Оба DBA
    - Backend-разработчик
  - **Желательно просмотреть:**
    - Тимлид / DevOps

## 5. Процесс

Перед демо MVP:

1. Каждая роль проходит по файлам, отмеченным как «Обязательное ревью».
2. Комментарии оставляются через PR или в виде встроенных заметок.
3. Ответственный за документацию обновляет как английскую, так и русскую версии, где это применимо.
