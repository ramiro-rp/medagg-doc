Руководство пользователя

В этом документе описано, как обычный пользователь (врач, исследователь, студент) работает с системой Medical Imaging Dataset Aggregator.

1. Вход в систему

  1. Откройте приложение в браузере.
    Пример: http://<server-host>/
  2. Если у вас уже есть аккаунт, нажмите «Войти».
  3. Если аккаунта нет, нажмите «Регистрация» и заполните форму.

Точные названия кнопок и полей зависят от реализации фронтенда. После завершения разработки замените плейсхолдеры реальными скриншотами.

2. Главный экран (Поиск)

После входа вы попадаете на страницу поиска. Обычно она содержит:

  • поле свободного текстового поиска;
  • фильтры по:
  • модальности;
  • анатомической области;
  • типу ML-задачи;
  • тегам;
  • таблицу с результатами поиска.

В навигации, как правило, есть разделы:

  • Search / Поиск;
  • My datasets / Мои наборы;
  • (для администратора) Admin / Админ.

3. Поиск наборов данных

3.1 Базовый поиск

  1. Введите запрос в строку поиска.
    Примеры:
  2. pneumonia chest xray
  3. пневмония грудная клетка КТ
  4. Задайте фильтры при необходимости:
  5. Модальность = X-ray / Рентген;
  6. Область = Chest / Грудная клетка;
  7. Тип задачи = Classification / Классификация.
  8. Нажмите Search / Поиск.

В таблице появятся наборы данных, удовлетворяющие запросу:

  • название, источник, модальность;
  • количество изображений;
  • оценка релевантности (если используется модуль поиска);
  • элементы управления для выбора наборов.

3.2 Просмотр деталей набора

Нажмите на строку набора или кнопку «Details / Подробнее», чтобы увидеть:

  • краткое описание;
  • ссылку на исходный ресурс;
  • список меток / патологий (если есть);
  • примерные изображения (если реализовано);
  • примерный размер набора.

4. Формирование собственной коллекции

4.1 Выбор наборов

На странице результатов поиска:

  1. Отметьте один или несколько наборов, которые хотите включить в коллекцию.
  2. Если интерфейс предоставляет дополнительные настройки, укажите их, например:
  3. максимальное число изображений из каждого набора;
  4. правила балансировки классов и т.п.

4.2 Запуск формирования

  1. Нажмите кнопку «Сформировать набор» (название может отличаться).
  2. Проверьте сводную информацию:
  3. сколько наборов выбрано;
  4. примерное число изображений;
  5. примерный размер архива.
  6. Подтвердите создание.

Система создаст запрос на формирование коллекции и перенаправит вас в раздел «Мои наборы» или покажет уведомление.

5. Раздел «Мои наборы» (история)

В разделе «Мои наборы» отображается список ранее сформированных коллекций.

Для каждой коллекции показываются:

  • идентификатор или название;
  • текст исходного запроса или краткое описание;
  • дата создания;
  • статус:
  • Pending / В процессе – коллекция ещё формируется;
  • Ready / Готов – архив доступен для скачивания;
  • Expired / Истёк – архив удалён, коллекцию можно пересоздать;
  • действия:
  • Download ZIP / Скачать (если статус Ready);
  • Regenerate / Пересоздать (если реализовано).

При нажатии Download ZIP скачивается архив, который обычно содержит:

  • каталог images/ с файлами изображений;
  • файл metadata.csv с таблицей метаданных по каждому изображению (ID, метки, модальность, исходный набор и т.п.).

6. Выход из системы

Нажмите на иконку профиля или пункт меню с вашим именем и выберите «Выход / Log out».

7. Частые проблемы и рекомендации

  • «Ничего не найдено»
  • Уберите часть фильтров или смягчите условия.
  • Проверьте, есть ли в каталоге наборы для выбранной модальности / задачи.
  • «Статус долго остаётся ‘В процессе’»
  • Возможна высокая нагрузка на сервер или медленная работа внешних источников.
  • Если ожидание значительно превышает нормальное время, обратитесь к администратору.
  • «Ссылка на скачивание не работает или архив истёк»
  • Коллекция могла быть автоматически удалена по истечении срока хранения. Попробуйте пересоздать набор или выполнить поиск заново.
